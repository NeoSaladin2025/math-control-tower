📝 클리닉 & 과제 자동화 로직

1 수업 중 질문 해결 (Clinic Stamp)

학생이 질문한 문제는 수업 시간에 선생님이 직접 해결해줍니다.
관리자 페이지에서 [해결 완료] 버튼을 누르면, 해당 문제에 '금일 해결(Clinic Resolved)' 스탬프가 찍힙니다.

DB Action: clinic_resolved_at = NOW() 업데이트

2 숙제 출제 시 자동 반영

선생님이 정규 과제를 출제할 때, '금일 해결된 문항'은 시스템이 자동으로 감지합니다.
"오늘 수업 때 배운 건 오늘 반드시 복습!" 원칙에 따라, 해당 문항들은 과제 리스트에 강제 포함됩니다.

  • 날짜 기준: 오늘(Today) 해결된 건만 불러옴.
  • 어제 해결한 건 내일 숙제에 포함되지 않음 (자동 소멸).
  • 학생 스스로 푼 문제(단순 정답)는 포함되지 않음. (선생님 손길이 닿은 것만!)

📤 정규 과제 출제 가이드

[teacher_assign_integrated.html] 페이지에서 수행합니다.

  1. 학생 선택: 과제를 부여할 학생을 선택합니다.
  2. 문제집 선택: 해당 학생이 보유한 문제집을 선택하면, 전체 풀이 현황(X-Ray Grid)이 뜹니다.
  3. 범위 설정: Start ~ End 번호를 입력합니다.
  4. 필터링 & 믹싱:
    • 1등급 / 서술형 / 일반 문항의 비율을 슬라이더로 조절합니다.
    • 복습(빨강)과 신규(파랑)의 비율을 조절합니다.
    • Tip: 새 문제는 1번부터 순차적으로 채워집니다. (학생들이 좋아해요!)
  5. 검수 및 전송: [출제 문항 검수하기] -> [최종 전송]을 누르면 학생 앱으로 즉시 전송됩니다.

🔒 자료 유출 방지 (Blob)

Blob URL이란?

실제 파일 경로(예: .../problems/001.webp)를 브라우저 메모리 가상 주소(예: blob:http://...)로 변환하여 보여주는 기술입니다.
사용자가 이미지 주소를 복사해도, 해당 세션이 끝나면 주소가 파기되므로 외부 공유가 불가능합니다.

// Eve's Security Code
const blob = await response.blob();
const url = URL.createObjectURL(blob);
imgElement.src = url; // 실제 주소 은폐 완료 ❤️

🛡️ Iframe 샌드박스

문제지 뷰어는 메인 페이지와 분리된 Iframe(격리 공간) 내부에서 실행됩니다.
우클릭, 드래그, 인쇄(Print) 기능을 강제로 차단하여 무단 복제를 막습니다.

  • contextmenu 차단: 우클릭 메뉴 안 뜸.
  • selectstart 차단: 텍스트/이미지 드래그 불가.
  • sandbox 속성: 스크립트 실행 제어.
